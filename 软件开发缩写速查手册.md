# 软件开发行业缩写速查手册

> 涵盖需求分析、开发、项目管理、测试、DevOps、数据库、接口开发、安全认证八大类

---

## 一、需求分析 (Business Analysis)

| 缩写 | 全称 | 中文 | 负责角色 |
|---|---|---|---|
| BRD | Business Requirement Document | 业务需求文档 | BA |
| FSD | Functional Specification Document | 功能规格说明书 | BA |
| TSD | Technical Specification Document | 技术设计文档 | Tech Lead |
| SRS | Software Requirement Specification | 软件需求规格说明 | BA |
| FRD | Functional Requirement Document | 功能需求文档 | BA |
| NFR | Non-Functional Requirement | 非功能需求(性能/安全/可用性等) | 架构师 |
| UC | Use Case | 用例 | BA |
| UAT | User Acceptance Testing | 用户验收测试 | BA/客户 |
| PRD | Product Requirement Document | 产品需求文档 | PM/产品经理 |
| RTM | Requirement Traceability Matrix | 需求跟踪矩阵 | BA |

**举例**: `The BRD needs sign-off before we draft the FSD.`

## 四、测试

| 缩写 | 全称 | 中文 |
|---|---|---|
| SIT | System Integration Testing | 系统集成测试 |
| UAT | User Acceptance Test | 用户验收 |
| FAT | Factory Acceptance Test | 工厂验收 |
| PAT | Production Acceptance Test | 生产验收 |
| E2E | End-to-End Test | 端到端测试 |
| UT | Unit Test | 单元测试 |
| Regression | Regression Test | 回归测试 |
| Smoke | Smoke Test | 冒烟测试 |
| TC | Test Case | 测试用例 |
| DoD | Definition of Done | 完成标准 |

**举例**: `Passed SIT. Waiting for UAT.`

> 💡 企业软件开发(D365/SAP等)中,测试链通常是: UT → SIT → UAT → PAT,层层验收后才能上生产环境。
